Template

#docs.hbs

Most pages on finn.no runs on the same .container > .page structure, these give the current widths to fit within the banners, and also the correct margins and responsive behaviour.

Basic template 
<!DOCTYPE html>
<html lang="no" prefix="og: http://ogp.me/ns#">
    <head>
        <meta charset="utf-8">
        <meta name="viewport" content="width=device-width, initial-scale=1">
        <meta http-equiv="X-UA-Compatible" content="IE=Edge">
        <title>FINN</title>
        <link rel="stylesheet" href="//static.finncdn.no/bb/latest/css/spaden/5.x/spaden.min.css">
        <!--[if IE 9]>
        <link rel="stylesheet" href="/static/ie9.e7628c1e.css">
        <![endif]-->
    </head>
    <body>
        <div class="container">
            <div class="page">
                <!-- page content -->
                <div class="line">
                    <div class="unit r-size2of3">
                        ...
                    </div>
                    <div class="unit r-size1of3">
                        ...
                    </div>
                </div>
            </div>
        </div>
    </body>
</html>

Grid system for all FINN pages

line

    ...
</div>

Groups units on one horizontal line. Note: for mobile layout units may be stacked to avoid horizontal scrolling.

unit 
<div class="line">
    <div class="unit">
        ...
    </div>
</div>

Base class which divides a line into sections (columns).

r-sizeXofY 
<div class="line">
    <div class="unit r-sizeXofY">
        ...
    </div>
</div>

Extends unit. Indicates the number of grid units this element spans, for example r-size2of3 would take up 2 of 3 columns. Collapses to 1 column at 768px.

Intended use: Main layout grid!

Margin/Padding

NB! these are absolute, if you need the kind of margin that collapses to something smaller on a mobile device, use .r-margin

Class names are built up by 3 letters in the following order:
[type][location][size]

type

p, m = padding, margin

location

a, t, r, b, l, h, v = all, top, right, bottom, left, horizontal, vertical

size

t, s, m, l, n = tiny(3px), small(5px), medium(10px), large(20px), none(0px)

some examples

pam = padding + all sides + medium
ptn = padding + top + none
mal = margin + all sides + large

<button class="mal">The button</button>

<button class="mam">The button</button>

<button class="pal mhs mtt mbl">The button</button>

Building tables

<table> <thead> <tbody> <tr> <th> <td> <colspan> <caption>

Tables are great for a lot of things. Great tables, however, need a bit of markup love to be useful, scalable, and readable (at the code level). Here are a few tips to help:

Always wrap your column headers in a <thead> such that hierarchy is <thead> > <tr> > <th>.

Similar to the column headers, all your table?s body content should be wrapped in a <tbody> so your hierarchy is <tbody> > <tr> > <td>.

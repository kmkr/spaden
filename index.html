This doc and organization

Spaden is devided into a core and a Component section, where the core pack contains a minimum base styling needed for most pages. The component section is a mixture of specific css to solve specific needs, and pure markup examples using classes from the core pack. The example section contains some of the examples/prototypes/proposals from the last couple of years, these may or may not be up to date and they are mostly kept for reference.

# docs.hbs

Highly customizable grid system used on all FINN pages. combine .line and multiple .unit to build complex responsive grids. There are several ways of controlling responsivness and also how many units you get pr line. (Think of line more like a containing block, sometimes its a line, sometimes elements may stack differently).

Property Description
.line Groups units on one horizontal line. Note: for mobile layout units may be stacked to avoid horizontal scrolling.
.unit Base class which divides a line into sections (columns).
.sizeXofY Extends unit. Indicates the fractional width of the unit, for example size3of4 would take up three quarters, or 75%, of the horizontal space.
Supported fractions:
  • 1
  • 1/2
  • 1/3
  • 2/3
  • 1/4
  • 3/4
  • 1/5
  • 2/5
  • 3/5
  • 4/5
  • 1/6
  • 5/6

Intended use: Non-responsive liquid elements

.r-sizeXofY Extends unit. Indicates the number of grid units this element spans, for example r-size2of3 would take up 2 of 3 columns. Collapses to 1 column at 768px.

Intended use: Column based contents
.r-colsXtoY Extends line. Responsive column setup for content. Classname indicates number of colums, for example r-cols2to4 will fit 2 columns on mobile screens and 4 columns from 768px and up.

Great for dynamic content when you dont know how many units youll get.

Supported combos: 1to2, 1to3, 1to4, 2to3, 2to4, 2to5, 2to6

Intended use: Column based contents

.colsXupto[Breakpoint]

Extends line.

Classname indicates number of colums at specific breakpoints, for example .cols1upto480 .cols2upto768 .cols4from990 will fit 1 column below 480px, 2 columns between 480px and 767px, and 4 columns from 990px to infinity

! Works in conjunction with colsXfrom990, remember to set this aswell, or 990px+ will get whatever is default (usually 1col )

.cols[X]upto[breakpoint] Sets number of columns up to specified breakpoint

.cols[X]from990 Sets number of columns from 990 and up

Supported column count: 1 - 8

Supported breakpoints: 480px, 640px, 768px, 990px

Intended use: Column based contents where we need more control

# docs.hbs

To use SVG icons, you have to insert an icons definition file in your markup. They are located in Spaden under dist/icons and grouped by type (e.g. market or share icons).

To refer to one of the icons, simply use an SVG <use></use> element. (Don't use the empty element <use/> because it fails in Safari 6.)

<table> <thead> <tbody> <tr> <th> <td> <colspan> <caption>

Tables are great for a lot of things. Great tables, however, need a bit of markup love to be useful, scalable, and readable (at the code level). Here are a few tips to help:

Always wrap your column headers in a <thead> such that hierarchy is <thead> > <tr> > <th>.

Similar to the column headers, all your table?s body content should be wrapped in a <tbody> so your hierarchy is <tbody> > <tr> > <td>.

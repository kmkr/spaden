# docs.hbs

Template

Most pages on finn.no runs on the same .container > .page structure, these give the current widths to fit within the banners, and also the correct margins and responsive behaviour.

Basic template 
<!DOCTYPE html>
<html lang="no" prefix="og: http://ogp.me/ns#">
    <head>
        <meta charset="utf-8">
        <meta name="viewport" content="width=device-width, initial-scale=1">
        <meta http-equiv="X-UA-Compatible" content="IE=Edge">
        <title>FINN</title>
        <link rel="stylesheet" href="//static.finncdn.no/bb/latest/css/spaden/5.x/spaden.min.css">
        <!--[if IE 9]>
        <link rel="stylesheet" href="/static/ie9.e7628c1e.css">
        <![endif]-->
    </head>
    <body>
        <div class="container">
            <div class="page">
                <!-- page content -->
                <div class="line">
                    <div class="unit r-size2of3">
                        ...
                    </div>
                    <div class="unit r-size1of3">
                        ...
                    </div>
                </div>
            </div>
        </div>
    </body>
</html>

You can start with this basic HTML template.

# docs.hbs

Grid

Highly customizable grid system used on all FINN pages

line 
<div class="line">
    ...
</div>

Groups units on one horizontal line. Note: for mobile layout units may be stacked to avoid horizontal scrolling.

unit 
<div class="line">
    <div class="unit">
        ...
    </div>
</div>

Base class which divides a line into sections (columns).

r-sizeXofY 
<div class="line">
    <div class="unit r-sizeXofY">
        ...
    </div>
</div>

Extends unit. Indicates the number of grid units this element spans, for example r-size2of3 would take up 2 of 3 columns. Collapses to 1 column at 768px.

Intended use: Main layout grid!

<div class="line">
    <div class="unit sizeXofY">
        ...
    </div>
</div>

  • 4/5
  • 1/6
  • 5/6

Extends unit. Indicates the fractional width of the unit, for example size3of4 would take up three quarters, or 75%, of the horizontal space.

Intended use: Non-responsive liquid elements

Extends line.

.cols[X]upto[breakpoint] Sets number of columns up to specified breakpoint

.cols[X]from990 Sets number of columns from 990 and up

Classname indicates number of colums at specific breakpoints, for example .cols1upto480 .cols2upto768 .cols4from990 will fit 1 column below 480px, 2 columns between 480px and 767px, and 4 columns from 990px to infinity

Intended use: Column based contents where we need more control

# docs.mustache

Mod

mod 
<div class="mod">
    <div class="inner">
        <div class="bd">
            ...
        </div>
    </div>
</div>
Property Description
.mod Main module wrapper.
.inner Layout container wrapper, this is the one to extend for custom boxes. Add predefined background color class (bg- prefix), and/or color class for text. See color definition for more info. In case your making a supercustom module this is the class to extend (background-image and so on).
.bd Content goes here
.shadow Extends .mod. Adds dropshadow and extra vertical padding, do not use on nested mods.
.listing Extends .mod. Use for listings like result listing etc. Adjusts vertical margins and hides the last hr
.clickable Extends .mod. Adds hover/focus states (the mod needs to be an a tag for this to work properly including keyboard support). Since the whole block has its own focus and hover states, it works well paired with .linkblock to disable link behavior for the actual content of the mod.

.green-skin Extends .mod. Color it green with a dark green frame.(Used for misc info boxes, boradcast etc.)

# spacing.mustache

Margin/Padding

NB! these are absolute, if you need the kind of margin that collapses to something smaller on a mobile device, use .r-margin

Class names are built up by 3 letters in the following order:
[type][location][size]

type

p, m = padding, margin

location

a, t, r, b, l, h, v = all, top, right, bottom, left, horizontal, vertical

size

t, s, m, l, n = tiny(3px), small(5px), medium(10px), large(20px), none(0px)

some examples

pam = padding + all sides + medium
ptn = padding + top + none
mal = margin + all sides + large

<button class="mal">The button</button>
<button class="mam">The button</button>
<button class="pal mhs mtt mbl">The button</button>

Ratio Description
.format1by1 Square format (equal sides).

Building tables

<table> <thead> <tbody> <tr> <th> <td> <colspan> <caption>

Tables are great for a lot of things. Great tables, however, need a bit of markup love to be useful, scalable, and readable (at the code level). Here are a few tips to help:

Always wrap your column headers in a <thead> such that hierarchy is <thead> > <tr> > <th>.

Similar to the column headers, all your table?s body content should be wrapped in a <tbody> so your hierarchy is <tbody> > <tr> > <td>.
